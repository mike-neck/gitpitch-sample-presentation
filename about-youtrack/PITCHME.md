#### クエリーを使ってチケットを探す

すべての基本はクエリーです。バーンダウンチャートやアジャイルボードといった機能はすべてクエリーによって作られていると言っても過言ではありません。

クエリーで覚えるべきポイントは二つです。

* キーとバリュー形式で指定する
* キーの値だけを覚えておけばよい

---

#### クエリーのポイント

##### キーとバリュー形式で指定する

条件を一つ指定する

```
# キー: バリュー
Asignee: shinya.mochida
```

条件を複数指定する(AND検索)

```
Asignee: shinya.mochida Status: Open
```

(`#{Assign to me}` とか `#UnResolved` などの特殊な条件もあります)

---

#### クエリーのポイント

##### キーの値だけを覚えておけばよい

* クエリーのキーの項目は、チケットのフィールド名が使われます
* フィールドにはチケットの状態・性質(分類)が記述されています
  * Type : チケットのタイプ(バグ/タスク)
  * State : 今の状態
  * Estimation : 見積もり
* このフィールドの値を元にプロジェクトを管理しているので、これらのフィールド名を把握しておくことはプロジェクト管理のベースである
* フィールドの値については YouTrack が補完してくれるから、覚えていなくてもよい(もちろん、フィールド名も補完してくれるのだが)

---

#### クエリーのポイント

##### 例外

* チケット同士の関連を表すクエリー
  * `Subtask of:` - `of` の後に指定されたチケットの子チケットを検索する(再帰的検索はできない)
  * `Relates to:` - `to` の後に指定されたチケットに関連するチケット など
* キーを指定しない - キーを省略できる
  * `#Task` - `Type: Task` と同意
  * `#UnResolved` - `Open`, `Submitted`, `In Progress` を合わせた状態
* ユーザーを指定するもの
  * `created by:`, `updated by:`, `commenter:`
* 活動日時を指定するもの
  * `commented:`, `updated:`, `resolved date:`
